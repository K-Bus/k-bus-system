# 요구사항 정의서 (K-Bus 고속버스 예매 시스템)

## 1. 프로젝트 개요
- **프로젝트명**: K-Bus 고속버스 예매 시스템
- **목적**: 콘솔 기반의 간단한 고속버스 좌석 예매/조회/취소 기능 제공
- **대상 사용자**: 일반 승객(사용자 ID 기반)
- **개발 범위**: Java SE 만으로 구현 ("이것이 자바다" 교재 범위 내: Collection, Stream API, File I/O 객체 직렬화, 정규표현식, 예외 처리, 열거형, 람다, 중첩 클래스 등)

## 2. 기능적 요구사항 (Functional Requirements)

| ID | 기능명 | 설명 |
|----|--------|------|
| FR-01 | 데이터 초기화 | 프로그램 시작 시 cities.txt / schedule.txt 파일을 읽어 노선·도시 정보를 로드한다. |
| FR-02 | 좌석/예약 데이터 로드 | 저장된 seats.ser, reservation.ser 파일을 읽어 기존 상태를 복원한다. (없으면 초기화) |
| FR-03 | 노선 검색 (출발지·도착지) | 사용자가 입력한 출발/도착 도시의 노선을 검색한다. |
| FR-04 | 노선 검색 (출발·도착·시간) | 출발/도착 + 출발 시각까지 포함하여 노선을 검색한다. |
| FR-05 | 좌석 조회 | 특정 노선의 전체 좌석 상태(예약/미예약)를 조회한다. |
| FR-06 | 예약 생성 | 지정한 좌석을 예약하고, 좌석/예약 내역 파일에 영속화한다. |
| FR-07 | 예약 조회 | 사용자 ID로 본인의 예약 내역을 조회한다. |
| FR-08 | 예약 취소 | 본인의 예약을 선택하여 취소하고, 좌석을 다시 미예약 상태로 되돌린다. |
| FR-09 | 종료 | 메뉴에서 종료 선택 시 자원 정리 후 프로그램을 종료한다. |

## 3. 비기능적 요구사항 (Non-functional Requirements)

| ID | 항목 | 내용 |
|----|------|------|
| NFR-01 | 영속성 | 좌석 및 예약 정보는 객체 직렬화(`ObjectOutputStream`)로 파일에 저장. |
| NFR-02 | 유지보수성 | 계층 분리 (Entry / Service / Model) 준수. |
| NFR-03 | 이식성 | 표준 JDK 11+ 에서만 동작, 별도 DB/네트워크 의존 없음. |
| NFR-04 | 신뢰성 | 파일 I/O, 사용자 입력 오류에 대해 예외 처리 전략 적용. |
| NFR-05 | 사용성 | 한국어 콘솔 메뉴, 잘못된 입력은 재입력 요청. |
| NFR-06 | 입력 검증 | 사용자 ID는 정규표현식(`^[a-zA-Z0-9]{2,10}$`)으로 검증. |
| NFR-07 | 응답성 | 모든 메뉴 동작은 1초 이내 응답 (콘솔 I/O 기준). |
| NFR-08 | 확장성 | 도시/노선 추가는 텍스트 파일 수정만으로 가능. |

## 4. 사용 기술 요구사항 (필수)
- **Collection**: `List`, `Map`, `ArrayList`, `HashMap` 을 사용해 노선/예약 관리
- **File I/O (객체 직렬화)**: `ObjectInputStream`, `ObjectOutputStream` 으로 상태 영속화
- **Stream API**: 노선·좌석·예약 검색/필터링에 Stream 사용
- **정규표현식**: 사용자 ID 유효성 검증
- **예외 처리 전략**: 체크 예외는 서비스 계층에서 래핑, 사용자 입력 오류는 `try-catch` 후 재입력
- **열거형(Enum)**: 메뉴 항목을 열거형으로 정의
- **람다/중첩 클래스**: Stream 내부에서 람다식 활용

## 5. 제약사항
- GUI 는 사용하지 않고 콘솔 기반으로 구현
- 모델 클래스의 getter/setter/생성자/toString 은 수동 작성
