# 시나리오 문서

## 시나리오 1. 예약 생성 (정상 흐름)

**행위자**: 사용자 `user01`

1. 프로그램을 실행한다.
2. 시스템이 cities.txt / schedule.txt / seats.ser / reservation.ser 를 로드한다.
3. 메뉴에서 `1. 예약하기` 를 선택한다.
4. 사용자 ID `user01` 입력 (정규식 통과).
5. 출발지 `서울(1)`, 도착지 `부산(2)` 입력.
6. 시스템이 서울→부산 노선 24건을 Stream 으로 필터링하여 출력.
7. 원하는 시간 `13:00~14:00` 선택 (busRouteId=14).
8. 좌석 현황을 출력, 사용자가 `5번 좌석` 선택.
9. 시스템이 좌석 예약 처리 → Reservation(id=1) 생성 → seats.ser + reservation.ser 에 직렬화 저장.
10. "예약이 완료되었습니다." 메시지 출력.

## 시나리오 2. 예약 조회 및 취소

**행위자**: 사용자 `user01`

1. 메뉴에서 `2. 예약확인` 을 선택한다.
2. 사용자 ID `user01` 입력.
3. 시스템이 reservation.ser 를 Stream 으로 필터링하여 본인 예약 1건 출력.
4. 취소할 예약 ID `1` 입력.
5. 시스템이 예약 제거, 좌석 해제, 파일 저장.
6. "예약이 취소되었습니다." 메시지 출력.

## 시나리오 3. 잘못된 입력 (예외 흐름)

1. 사용자가 ID 로 `!!!` 입력.
2. 정규식 `^[a-zA-Z0-9]{2,10}$` 검증 실패 → "올바른 ID 가 아닙니다" 출력 후 재입력 요청.
3. 출발지로 `99` 입력 → 존재하지 않는 도시 → "잘못된 도시 번호" 출력 후 재입력.
4. 이미 예약된 좌석 선택 → "이미 예약된 좌석입니다" 출력 후 재입력.
