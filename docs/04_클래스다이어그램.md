# 클래스 다이어그램 설명

> 원본 PDF: `고속버스 예매시스템 클래스 다이어그램.pdf` 참조

## 1. 패키지 구성

```
src/
├── Main.java                        (Entry)
├── model/                           (VO / 엔터티)
│   ├── User.java
│   ├── BusRoute.java
│   ├── Seat.java
│   └── Reservation.java
└── service/                         (비즈니스 로직)
    ├── FileIOService.java
    ├── SearchService.java
    └── ReservationService.java
```

## 2. 계층 의존 관계

```
  Entry ──사용──▶  Service ──참조──▶  Model
                     │
                     └──read / save──▶  FileIOService ──▶ data/*.txt / *.ser
```

- Main 은 SearchService, ReservationService 를 호출한다.
- SearchService, ReservationService 는 FileIOService 를 통해 파일을 읽고 쓴다.
- 모든 서비스는 Model 객체(User, BusRoute, Seat, Reservation)를 참조한다.

## 3. Model 클래스

### User
| 필드 | 타입 | 설명 |
|------|------|------|
| userId | String | 사용자 ID (정규식 검증) |

### BusRoute
| 필드 | 타입 | 설명 |
|------|------|------|
| busRouteId | int | 노선 식별자 (PK) |
| departure | int | 출발 도시 ID |
| destination | int | 도착 도시 ID |
| seats | Seat[] | 좌석 배열 (포함 관계) |
| departureTime | String | 출발 시각 (예 "13:00~14:00") |

### Seat
| 필드 | 타입 | 설명 |
|------|------|------|
| seat | boolean | 예약 여부 (true = 예약됨) |

### Reservation
| 필드 | 타입 | 설명 |
|------|------|------|
| id | int | 예약 ID (자동 증가) |
| userId | String | 예약자 ID |
| busRouteId | int | 노선 ID |
| date | String | 예약 일자 (yyyy-MM-dd) |

## 4. Service 클래스

### FileIOService
| 메서드 | 반환 | 설명 |
|--------|------|------|
| readCities() | `Map<Integer,String>` | cities.txt 로드 |
| readSchedule() | `List<BusRoute>` | schedule.txt 로드 |
| readSeats() | `Map<Integer,Seat[]>` | seats.ser 역직렬화 |
| readReservation() | `List<Reservation>` | reservation.ser 역직렬화 |
| writeCities(map) | void | cities.txt 저장 |
| writeSchedule(list) | void | schedule.txt 저장 |
| writeSeats(map) | void | seats.ser 직렬화 |
| writeReservation(list) | void | reservation.ser 직렬화 |

### SearchService
| 메서드 | 반환 | 설명 |
|--------|------|------|
| searchBusRoute(departure, destination) | `List<BusRoute>` | Stream 필터로 노선 검색 |
| searchBusRoute(departure, destination, time) | `List<BusRoute>` | 시간 조건 추가 필터 |
| checkSeats(busRouteId) | `Seat[]` | 좌석 현황 조회 |

### ReservationService
| 메서드 | 반환 | 설명 |
|--------|------|------|
| makeReservation(userId, busRoute, seatNo) | `Reservation` | 좌석 예약, 파일 저장 |
| checkReservation(userId) | `List<Reservation>` | 본인 예약 조회 (Stream) |
| cancelReservation(reservation) | boolean | 예약 취소, 좌석 해제, 파일 저장 |

## 5. 자료구조 / 디자인 패턴

- **자료구조**
  - `HashMap<Integer,String>`: 도시 ID → 이름 매핑 (O(1) 조회)
  - `ArrayList<BusRoute>`: 노선 목록, 순차 탐색 + Stream 필터
  - `HashMap<Integer,Seat[]>`: 노선 ID → 좌석 배열 매핑 (좌석 O(1) 접근)
  - `ArrayList<Reservation>`: 예약 내역

- **디자인 패턴**
  - **Singleton 유사**: FileIOService 는 Main 에서 단일 인스턴스로 유지
  - **Layered Architecture**: Entry → Service → Model 3 계층 분리
  - **DTO/VO**: Model 패키지 객체는 상태만 보유하는 VO 역할
