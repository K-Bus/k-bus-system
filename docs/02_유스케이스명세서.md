# 유스케이스 다이어그램 및 명세서

## 1. 유스케이스 다이어그램

```
                    ┌──────────────────────────────────┐
                    │       K-Bus 예매 시스템            │
                    │                                  │
                    │   (UC-01) 노선 검색               │
   ┌──────┐         │   (UC-02) 좌석 조회               │
   │ 사용자 │────────▶│   (UC-03) 예약 생성               │
   └──────┘         │   (UC-04) 예약 조회               │
                    │   (UC-05) 예약 취소               │
                    │                                  │
                    └──────────────────────────────────┘
```

- **Actor**: 사용자 (User)
- **시스템 경계**: 콘솔 프로그램 내부 (Main + Service + Model)

## 2. 유스케이스 목록

| ID | 유스케이스 | Actor | 설명 |
|----|-----------|-------|------|
| UC-01 | 노선 검색 | 사용자 | 출발·도착(·시간) 조건으로 노선을 검색 |
| UC-02 | 좌석 조회 | 사용자 | 특정 노선의 좌석 점유 상태 확인 |
| UC-03 | 예약 생성 | 사용자 | 좌석을 선택하여 예약 |
| UC-04 | 예약 조회 | 사용자 | 본인 ID로 예약 내역 조회 |
| UC-05 | 예약 취소 | 사용자 | 예약을 삭제하고 좌석을 해제 |

---

## 3. 유스케이스 명세서

### UC-01. 노선 검색
| 항목 | 내용 |
|------|------|
| **유스케이스 ID** | UC-01 |
| **유스케이스명** | 노선 검색 |
| **Actor** | 사용자 |
| **개요** | 출발·도착 도시(와 시간)로 운행 중인 노선을 검색 |
| **전제조건** | cities.txt, schedule.txt 가 정상 로드된 상태 |
| **기본 흐름** | 1. 사용자가 "예약하기" 선택<br>2. 출발/도착 도시 입력<br>3. 시스템이 Stream API 로 노선 필터링<br>4. 결과 리스트를 출력 |
| **대안 흐름 (시간 조건 포함)** | 2-1. 출발 시각을 추가로 입력<br>3-1. 출발 시각과 일치하는 노선만 필터링 |
| **예외 흐름** | - 존재하지 않는 도시 입력 → 오류 메시지 후 재입력<br>- 검색 결과 0건 → "노선이 없습니다" 출력 |
| **후조건** | 노선 리스트가 화면에 출력됨 |

### UC-02. 좌석 조회
| 항목 | 내용 |
|------|------|
| **유스케이스 ID** | UC-02 |
| **Actor** | 사용자 |
| **전제조건** | 유효한 busRouteId 가 선택된 상태 |
| **기본 흐름** | 1. 사용자가 노선을 선택<br>2. 시스템이 seats.ser 의 해당 노선 좌석 배열을 조회<br>3. 좌석 번호별 예약 여부를 출력 |
| **예외 흐름** | busRouteId 가 존재하지 않으면 예외 처리 후 메뉴로 복귀 |
| **후조건** | 좌석 상태가 출력됨 |

### UC-03. 예약 생성
| 항목 | 내용 |
|------|------|
| **유스케이스 ID** | UC-03 |
| **Actor** | 사용자 |
| **전제조건** | 노선/좌석이 선택되고, 사용자 ID 가 정규식에 부합 |
| **기본 흐름** | 1. 사용자가 좌석 번호 입력<br>2. 시스템이 좌석 미예약 여부 확인<br>3. Reservation 객체 생성 (id 자동 부여)<br>4. 좌석을 예약 처리 (true)<br>5. seats.ser, reservation.ser 에 저장<br>6. 예약 성공 메시지 출력 |
| **예외 흐름** | - 이미 예약된 좌석 → "이미 예약됨" 후 재입력<br>- 파일 쓰기 실패 → 롤백 후 오류 메시지 |
| **후조건** | 예약이 영속 저장되고 좌석 상태가 갱신됨 |

### UC-04. 예약 조회
| 항목 | 내용 |
|------|------|
| **유스케이스 ID** | UC-04 |
| **Actor** | 사용자 |
| **전제조건** | 유효한 사용자 ID |
| **기본 흐름** | 1. 사용자 ID 입력<br>2. 시스템이 Stream 으로 해당 사용자 예약 필터링<br>3. 결과 출력 |
| **예외 흐름** | 예약 없음 → 안내 메시지 |
| **후조건** | 예약 내역이 화면에 출력됨 |

### UC-05. 예약 취소
| 항목 | 내용 |
|------|------|
| **유스케이스 ID** | UC-05 |
| **Actor** | 사용자 |
| **전제조건** | UC-04 로 본인 예약 목록이 조회된 상태 |
| **기본 흐름** | 1. 취소할 예약 id 입력<br>2. 시스템이 예약 객체 제거<br>3. 해당 좌석을 false 로 변경<br>4. 파일에 저장 |
| **예외 흐름** | 존재하지 않는 id 입력 시 안내 메시지 |
| **후조건** | 예약 삭제, 좌석 해제, 파일 갱신 |
