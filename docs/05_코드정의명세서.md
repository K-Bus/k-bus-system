# 코드 정의 명세서

> 모든 모델 클래스는 직렬화를 위해 `java.io.Serializable` 을 구현한다.
> 모든 getter/setter/생성자/toString 은 수동 작성한다.

---

## 1. model/User.java

| 항목 | 내용 |
|------|------|
| 패키지 | model |
| 구현 인터페이스 | Serializable |
| 필드 | `private String userId;` |
| 생성자 | `User()`, `User(String userId)` |
| 메서드 | `getUserId()`, `setUserId(String)`, `toString()` |

## 4. model/Reservation.java

| 항목 | 내용 |
|------|------|
| 패키지 | model |
| 구현 인터페이스 | Serializable |
| 필드 | `int id`, `String userId`, `int busRouteId`, `String date`, `int seatNumber` |
| 생성자 | 전체 필드 생성자, 기본 생성자 |

> 좌석 취소 시 해제할 좌석을 알아야 하므로 확장 필드로 정의한다.

## 5. service/FileIOService.java

| 메서드 시그니처 | 동작 |
|----------------|------|
| `Map<Integer,String> readCities()` | cities.txt 를 줄 단위로 읽어 `Map<id,도시명>` 반환 |
| `List<BusRoute> readSchedule()` | schedule.txt 를 줄 단위로 읽어 `List<BusRoute>` 반환 (seats 는 null) |
| `Map<Integer,Seat[]> readSeats()` | seats.ser 역직렬화. 파일 없으면 빈 Map 반환 |
| `List<Reservation> readReservation()` | reservation.ser 역직렬화. 파일 없으면 빈 List |
| `void writeCities(Map<Integer,String>)` | cities.txt 텍스트 저장 |
| `void writeSchedule(List<BusRoute>)` | schedule.txt 텍스트 저장 |
| `void writeSeats(Map<Integer,Seat[]>)` | seats.ser 직렬화 저장 |
| `void writeReservation(List<Reservation>)` | reservation.ser 직렬화 저장 |

## 6. service/SearchService.java

| 메서드 | 동작 |
|--------|------|
| `List<BusRoute> searchBusRoute(int departure, int destination)` | Stream 필터 : `r -> r.departure==dep && r.destination==dest` |
| `List<BusRoute> searchBusRoute(int departure, int destination, String time)` | 위 조건 + `r.departureTime.equals(time)` |
| `Seat[] checkSeats(int busRouteId)` | seatsMap 에서 해당 노선 좌석 반환. 없으면 `null` |

## 7. service/ReservationService.java

| 메서드 | 동작 |
|--------|------|
| `Reservation makeReservation(String userId, BusRoute busRoute, int seatNo)` | 좌석 true 로 변경, Reservation(id 자동증가) 생성, seats+reservation 저장. 이미 예약된 좌석이면 `IllegalStateException`. |
| `List<Reservation> checkReservation(String userId)` | Stream 필터로 본인 예약 반환 |
| `boolean cancelReservation(Reservation reservation)` | 예약 제거 + 좌석 false + 파일 저장. 성공 여부 반환 |

## 9. 예외 처리 전략

| 예외 상황 | 처리 방법 |
|-----------|----------|
| `IOException` (파일 I/O) | 서비스 계층에서 `RuntimeException` 으로 래핑 |
| `ClassNotFoundException` (역직렬화) | 동일하게 래핑 |
| 잘못된 메뉴 입력 | `try-catch (NumberFormatException)` + 재입력 |
| 중복 좌석 예약 | `IllegalStateException` 후 Main 에서 catch, 메시지 출력 |
| 존재하지 않는 예약 id 취소 | 메서드가 `false` 반환, Main 에서 안내 메시지 |
